title: AWS Bedrock
description: Route Anthropic models through AWS Bedrock for enterprise deployments.

## Setup

```bash
pnpm add @anthropic-ai/bedrock-sdk
```

```typescript
import { BedrockAnthropicProvider } from "noumen/bedrock";

const provider = new BedrockAnthropicProvider({
  region: "us-east-1",                                     // default
  model: "us.anthropic.claude-sonnet-4-v1:0",               // default
});
```

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `region` | `string` | `"us-east-1"` | AWS region |
| `model` | `string` | `"us.anthropic.claude-sonnet-4-v1:0"` | Default model in Bedrock format |
| `credentials` | `object` | — | Explicit AWS credentials (`accessKeyId`, `secretAccessKey`, optional `sessionToken`). Falls back to the default credential chain when omitted. |
| `baseURL` | `string` | — | Custom base URL for a Bedrock-compatible endpoint |
| `cacheControl` | `CacheControlConfig` | — | Enable prompt caching |
| `client` | `AnthropicBedrock` | — | Pre-constructed client instance. When provided, all other connection options are ignored. |

## Authentication

When `credentials` is omitted, the SDK uses the standard AWS credential chain (environment variables, `~/.aws/credentials`, IAM roles, etc.).

For explicit credentials:

```typescript
const provider = new BedrockAnthropicProvider({
  region: "us-west-2",
  credentials: {
    accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
    secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
    sessionToken: process.env.AWS_SESSION_TOKEN,
  },
});
```

## Prompt caching

Enable prompt caching to reduce costs on repeated system prompts:

```typescript
const provider = new BedrockAnthropicProvider({
  cacheControl: { enabled: true },
});
```

## Streaming

The Bedrock provider uses the shared Anthropic streaming layer. It maps Bedrock's response events to noumen's OpenAI-compatible chunk format identically to the direct Anthropic provider.

## Models

Use Bedrock model IDs. Common choices:

- `us.anthropic.claude-sonnet-4-v1:0` -- balanced speed and capability
- `us.anthropic.claude-opus-4-v1:0` -- highest capability
- `us.anthropic.claude-haiku-3-v1:0` -- fastest, most affordable
